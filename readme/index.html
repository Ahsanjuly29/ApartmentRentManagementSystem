Installation

Follow these steps to set up the HRMS on your local machine:

                1. Clone the Repository: git clone https://github.com/your-repo/HRMS.git
                2. Navigate into the Project Directory: cd HRMS
                3. Install Dependencies: composer install
                4. Configure the Environment: cp .env.example .env
                5. Generate Application Key: php artisan key:generate
                6. Run Migrations: php artisan migrate
                7. Serve the Application: php artisan serve

Database Migrations

The project includes the following database tables:

  • floors - Stores floor details for the building.
  • apartments - Contains apartment information linked to floors.
  • renters - Stores renter profiles and unique access codes.
  • service_charges - Logs dynamic service charges for each apartment.
  • bills - Holds rent and service charge bill information.
  • payments - Tracks payments made by renters.
  • bank_info - Records renter's bank payment details.

Features

  • Manage multiple floors and apartments.
  • Assign renters to one or more apartments.
  • Dynamic service charge assignment for each apartment.
  • Renters can view, download bills in PDF format.
  • Store and use renter's bank payment details.
  • Track payment history and outstanding amounts.

Admin Interface

The admin dashboard includes tools to manage floors, apartments, bills, and renters efficiently. Admins can:

  • Manage floors and apartments
  • Assign renters to apartments
  • Generate monthly bills and track due payments
  • Update dynamic service charges

Client View

Property managers can track rental earnings, view reports, and monitor payment statuses from the client dashboard. The dashboard allows:

  • Managing apartments and renters
  • Generating reports on total rent and expenses
  • Tracking overdue payments

Renter View

Renters can access their bills and payment history using a unique code. Key features include:

  • Unique access code login for bill viewing
  • View, print, and download bills in PDF format
  • Save bank payment information
  • View full payment history and outstanding balances

API Endpoints (Optional)

The application provides optional API endpoints for external integrations:

  • GET /api/renter/{id}/bills - Retrieve all bills for a renter.
  • POST /api/renter/{id}/payment - Submit payment information.
  • GET /api/apartment/{id}/service-charges - Retrieve service charges for an apartment.
